# Change Log - 28-03-2026
# Author: Sunny
## Fixes and Enhancements

- **Email & Phone Format Validation**: Reverted from real-time existence checking (due to API unreliability) to strict format validation using **Zod**.
    - **Email Validation**: Ensures the email follows the standard `user@domain.com` pattern.
    - **Phone Validation**: Added a regex check (`/^\+?[0-9\s\-()]{7,15}$/`) to ensure the phone number is in a valid international or local format.
    - **Required Fields**: Kept "First Name" and "Last Name" as mandatory fields to ensure lead data quality.
- **CRM Accounts Validation**: Added the same strict format validation (Email & Phone) to the **Accounts** page.
    - **Account Name**: Now a required field.
    - **Email & Phone Validation**: Matching the same patterns used in the Leads module.
- **CRM Contacts Validation**: Added the same strict format validation (Email & Phone) to the **Contacts** page.
    - **First & Last Name**: Now mandatory fields.
    - **Email & Phone Validation**: Matching the same patterns used in the Leads and Accounts modules.
- **CRM Opportunities Validation**: Added strict validation for the **Opportunities** page.
    - **Opportunity Name**: Now a required field.
    - **Amount**: Validated as a non-negative number.
    - **Probability**: Validated to be between 0 and 100%.
    - **Modified**: `src/modules/crm/pages/OpportunitiesPage.tsx`

- **Quote Builder Fix**: Resolved an issue where some accounts were not showing up in the "Create Quote" screen.
    - **Inclusion**: Switched from a restrictive manual query to the standard `useAccounts` CRM hook. This ensures all accessible accounts are now visible for quoting.
    - **Address Fix**: Updated the address field mapping to correctly display "Bill To" info from the account.
    - **Modified**: `src/modules/cpq/pages/QuoteBuilderPage.tsx`

- **CPQ Quote Workflow UX Fix**: Prevented invalid quote status actions from reaching users.
    - **Edit Guardrail**: Only draft quotes now expose the `Edit` action from the quotes list.
    - **Route Protection**: Directly opening the edit page for a non-draft quote now shows a clear message and routes the user back to the quote instead of failing with a status-transition error.
    - **Valid Actions Only**: Removed the invalid `draft -> sent` action from the quotes list and added the valid `approved -> sent` action to the quote detail page.
    - **Status Safety**: Same-status quote updates are now treated as no-ops instead of throwing unnecessary transition errors.
    - **Modified**: `src/modules/cpq/hooks/useCPQ.ts`, `src/modules/cpq/pages/QuoteBuilderPage.tsx`, `src/modules/cpq/pages/QuotesListPage.tsx`, `src/modules/cpq/pages/QuoteDetailPage.tsx`

- **Quote to Contract Prefill Fix**: Fixed the CLM conversion flow so approved quotes now carry their details into the contract form.
    - **Scoped Quote Loading**: Replaced the owner-only conversion query with the standard scoped quote hook, so the convert screen can read the same quote data visible on the quote detail page.
    - **Contract Details Prefill**: Account, contact, contract name, value, quote link, and related opportunity are now prefilled correctly when creating a contract from a quote.
    - **Fallback UX**: Added a clear warning state when a quote-linked contract form opens without being able to load the source quote.
    - **Modified**: `src/modules/clm/pages/ContractBuilderPage.tsx`

- **Legacy Dashboard Query Preservation Fix**: Fixed legacy redirects so they no longer drop query parameters during navigation.
    - **Root Cause**: The quote detail page opens the contract form through the legacy `/dashboard/...` path, and the redirect into the tenant workspace was removing `?quote_id=...`.
    - **Conversion Flow Fix**: The `Convert to Contract` action now keeps the quote ID during redirect, so the contract builder receives the source quote and can prefill its fields.
    - **Coverage**: Applied the same query/hash preservation to legacy dashboard, portal, and admin redirects for consistent behavior.
    - **Modified**: `src/app/App.tsx`

- **Contract Create UUID Payload Fix**: Fixed contract creation errors caused by optional fields being submitted as empty strings.
    - **Root Cause**: Optional UUID and date fields such as `template_id`, `opportunity_id`, and blank dates were being sent to Supabase as `""`, which PostgreSQL rejects for UUID/date columns.
    - **Submit for Review Fix**: Contract create now normalizes blank optional UUID/date fields to `null` before insert, so `Submit for Review` and `Save as Draft` no longer fail when optional fields are left empty.
    - **Edit Safety**: Applied the same normalization to contract updates so the edit flow does not reintroduce the same issue later.
    - **Modified**: `src/modules/clm/hooks/useCLM.ts`

- **Contract Draft Edit Workflow Fix**: Fixed the contract edit flow so only draft contracts can be edited and the edit form now loads existing values correctly.
    - **Edit Prefill**: The `/clm/contracts/:id/edit` route now loads the existing contract into the builder instead of opening a blank create form.
    - **Draft-Only Editing**: Removed `Edit` actions for approved and other non-draft contracts from the contracts list and contract detail quick actions.
    - **Route Guardrail**: Directly opening the edit page for a non-draft contract now shows a clear message and routes the user back to the contract instead of allowing invalid edits.
    - **Modified**: `src/core/types/clm.ts`, `src/modules/clm/pages/ContractBuilderPage.tsx`, `src/modules/clm/pages/ContractsListPage.tsx`, `src/modules/clm/pages/ContractDetailPage.tsx`

- **Contract Action Menu Edit Restore**: Restored the `Edit` option in the contracts list action menu.
    - **Menu Visibility**: The contracts list dropdown now always shows `Edit` again for easier discovery.
    - **Guardrail Kept**: Non-draft contracts are still protected by the edit-page guard, so approved contracts do not reopen as editable records.
    - **Modified**: `src/modules/clm/pages/ContractsListPage.tsx`

- **Contract Send-for-Signature Route Fix**: Fixed the contracts workflow so `Send for Signature` no longer opens a 404 page.
    - **Root Cause**: Contract actions were linking to `/clm/contracts/:id/sign`, but the actual e-sign route is `/clm/esign/:id`.
    - **Action Fix**: Updated the contracts list and contract detail actions to open the correct CLM e-sign page directly.
    - **Compatibility Redirect**: Added a router redirect so older `/clm/contracts/:id/sign` links still land on the e-sign screen.
    - **Modified**: `src/app/App.tsx`, `src/modules/clm/pages/ContractsListPage.tsx`, `src/modules/clm/pages/ContractDetailPage.tsx`
