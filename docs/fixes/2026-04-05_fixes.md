# Fixes - 2026-04-05

## 1. Activity Log "Session Agent" Resolution

**Issue:**
The Activity Log in the Organization Performance tab was displaying "System User" for all actions, or "Agent" for non-owners, regardless of who performed the action.

**Root Causes:**
1. **Frontend Logic**: The `useOrganizationPerformance` hook was not fetching user profile data and relied solely on metadata which was often null.
2. **Database Security (RLS)**: The `profiles` table had a restrictive Row Level Security (RLS) policy that prevented organization owners from seeing their employees' names—they could only see their own.

**Implemented Fixes:**
1. **Hook Update**: 
   - Modified `useOrganizationPerformance.ts` to perform a two-step fetch.
   - It now retrieves `profiles` (for full names) and `organization_memberships` (for emails) for all users in the logs.
   - Implemented a prioritized name resolution: `Full Name` > `Email Prefix` > `Metadata` > `System User`.
2. **Database Migration (`063_modify_profiles_select_policy.sql`)**:
   - Dropped the restrictive `profiles_select` policy.
   - Replaced it with a new policy that allows users within the same organization to see each other's profiles, provided they are active members.

**Outcome:**
Organization Owners and members can now correctly identify which agent performed specific actions in the Activity Log by their actual names or email identifiers.

## 2. Global Full Name Resolution (Admin Panel Greeting)

**Issue:**
The Organization Admin Dashboard and Top Bar were displaying "sahilk7rajput" (email prefix) instead of the user's actual name.

**Root Causes:**
1. **Context Limitation**: The core `AuthContext` only shared the `User` and `Session` objects but didn't fetch the corresponding `profiles` data.
2. **Frontend Logic**: UI components were manually splitting the email as a primary source of name display.

**Implemented Fixes:**
1. **Core Auth Update**:
   - Updated `auth-context.ts` to include `fullName`.
   - Modified `AuthProvider.tsx` to automatically fetch the `full_name` from the `profiles` table whenever the session is established or changed.
2. **Admin UI Update**:
   - Modified `OrganizationAdminDashboard.tsx` to greet the user by their full name (e.g., "Hello, Sahil Rajput").
   - Modified `AdminTopBar.tsx` to use the same full name for the profile menu and generated initials.

**Outcome:**
All authenticated users now have their real names available globally via `useAuth()`, and the Admin Panel correctly displays these names, providing a more professional and personal experience.
