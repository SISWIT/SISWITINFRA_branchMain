# Fixes - 2026-04-09

**Author:** Sunny

## 1. Admin Dashboard Initial Metric Loading Delays

**Issue:**
The Organization Admin "Command Center" dashboard could surface `0` values for key metrics during the first load, creating the impression that the organization had no leads, contracts, or pending invitations.

**Root Cause:**
1. `OrganizationAdminDashboard.tsx` did not provide metric-specific skeletons during the initial hydration phase.
2. `useOrganizationAdminDashboard.ts` silently converted Supabase query failures into fallback `0` KPI values because count and list query errors were not being checked before building the dashboard payload.

**Implemented Fixes:**
1. Added dashboard skeleton placeholders for the hero, KPI cards, activity feed, chart, and table sections during the initial load.
2. Added explicit Supabase error checks inside `useOrganizationAdminDashboard.ts` so failed queries throw instead of being interpreted as valid zero metrics.
3. Enabled `keepPreviousData` behavior with a `staleTime` window so previously loaded dashboard data remains stable during refreshes.
4. Removed an unused dashboard memberships count query from the admin dashboard hook to reduce unnecessary request work.
5. Added a retry state in the admin dashboard UI for failed data synchronization.
6. Corrected the pending invitation count query to filter the `tenant_invitations` view by `tenant_id`, which matches the actual view schema.

**Files Updated:**
- `src/workspaces/organization_admin/pages/OrganizationAdminDashboard.tsx`
- `src/workspaces/organization_admin/hooks/useOrganizationAdminDashboard.ts`

**Outcome:**
The Command Center now shows loading skeletons while metrics hydrate and avoids presenting transient or error-driven `0` KPI values as if they were final data.
