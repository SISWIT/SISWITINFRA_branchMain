## 1. Admin Dashboard Initial Metric Loading Delays

**Issue:**
The Organization Admin "Command Center" dashboard could surface `0` values for key metrics during the first load, creating the impression that the organization had no leads, contracts, or pending invitations.

**Root Cause:**
1. `OrganizationAdminDashboard.tsx` did not provide metric-specific skeletons during the initial hydration phase.
2. `useOrganizationAdminDashboard.ts` silently converted Supabase query failures into fallback `0` KPI values because count and list query errors were not being checked before building the dashboard payload.

**Implemented Fixes:**
1. Added dashboard skeleton placeholders for the hero, KPI cards, activity feed, chart, and table sections during the initial load.
2. Added explicit Supabase error checks inside `useOrganizationAdminDashboard.ts` so failed queries throw instead of being interpreted as valid zero metrics.
3. Enabled `keepPreviousData` behavior with a `staleTime` window so previously loaded dashboard data remains stable during refreshes.
4. Removed an unused dashboard memberships count query from the admin dashboard hook to reduce unnecessary request work.
5. Added a retry state in the admin dashboard UI for failed data synchronization.
6. Corrected the pending invitation count query to filter the `tenant_invitations` view by `tenant_id`, which matches the actual view schema.

**Files Updated:**
- `src/workspaces/organization_admin/pages/OrganizationAdminDashboard.tsx`
- `src/workspaces/organization_admin/hooks/useOrganizationAdminDashboard.ts`

**Outcome:**
The Command Center now shows loading skeletons while metrics hydrate and avoids presenting transient or error-driven `0` KPI values as if they were final data.

**Verification:**
- File lint passed:
  - `npx eslint src/workspaces/organization_admin/pages/OrganizationAdminDashboard.tsx src/workspaces/organization_admin/hooks/useOrganizationAdminDashboard.ts`
- Manual test passed:
  - Open the Command Center dashboard from a fresh load
  - Confirm skeleton placeholders appear while data is loading
  - Confirm KPI cards no longer flash misleading `0` values before the real counts arrive

## 2. TypeScript Config Deprecation Warning

**Issue:**
The editor reported a TypeScript configuration warning that `baseUrl` is deprecated and will stop functioning in TypeScript 7.0.

**Root Cause:**
Both `tsconfig.app.json` and the root `tsconfig.json` still declared `baseUrl`, even though the project only needs the `paths` alias mapping for `@/*`.

**Implemented Fixes:**
1. Removed `baseUrl` from `tsconfig.app.json`.
2. Removed `baseUrl` from the root `tsconfig.json`.
3. Kept the existing `paths` alias mapping intact so `@/*` imports continue to resolve correctly.
4. Removed an unused `Activity` import from `OrganizationAdminDashboard.tsx` that surfaced while validating the config cleanup.

**Files Updated:**
- `tsconfig.app.json`
- `tsconfig.json`
- `src/workspaces/organization_admin/pages/OrganizationAdminDashboard.tsx`

**Outcome:**
The `baseUrl` deprecation warning is resolved without changing the project’s import alias behavior.

**Verification:**
- File lint passed:
  - `npx eslint src/workspaces/organization_admin/pages/OrganizationAdminDashboard.tsx src/workspaces/organization_admin/hooks/useOrganizationAdminDashboard.ts`
- TypeScript validation:

## 4. Admin Report 2.3 Navigation Delay / "Double Click" Requirement

**Issue:**
Sidebar navigation in the Organization Admin "Command Center" could feel unresponsive, especially when opening heavier module routes like CRM or CLM. The click sometimes appeared not to register until a second attempt or until the main content changed much later.

**Root Cause:**
1. The admin workspace relied on the app-level lazy-route `Suspense` boundary, so slower route loads could make the full experience feel stalled instead of showing a content-area transition immediately.
2. The admin sidebar only reflected route activation after React Router finished navigation, which left no instant visual confirmation when a link was clicked.

**Implemented Fixes:**
1. Added an admin-specific `Suspense` fallback inside `TenantAdminLayout` so route loading is handled within the main content area instead of feeling like the whole workspace has frozen.
2. Added optimistic pending-link feedback in `AdminSidebar` so the clicked destination highlights immediately while navigation is still settling.
3. Kept the short route-transition overlay in the content area so page changes feel deliberate and responsive.

**Files Updated:**
- `src/workspaces/organization_admin/layout/TenantAdminLayout.tsx`
- `src/workspaces/organization_admin/components/AdminSidebar.tsx`

**Outcome:**
Admin sidebar navigation now gives immediate visual feedback, and lazy route loading stays localized to the content region to reduce the perception that links require a second click.
