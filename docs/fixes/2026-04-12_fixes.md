## 2. Bug Fix: `dataset` Type Error in Reveal Observer (Passed)

**Date:** 2026-04-12  
**Status:** PASSED

**Issue:**  
TypeScript error: `Property 'dataset' does not exist on type 'Element'.`

**Root Cause:**  
`IntersectionObserverEntry.target` is typed as `Element`, so direct `dataset` access is not type-safe without narrowing.

**Implemented Fixes:**
- Added runtime/type narrowing (`entry.target instanceof HTMLElement`) before reading `dataset.revealId`.
- Kept observer behavior unchanged while making the code type-safe.

**Files Updated:**
- `src/workspaces/website/pages/Products.tsx`

**Outcome:**

## 9. Bug Fix: ERP Create Fails Due to Missing Organization/Tenant Context (Passed)

**Date:** 2026-04-12  
**Status:** PASSED

**Issue:**  
Creating ERP records from UI pages failed at API layer for:
- Stock Item
- Purchase Order
- Production Order
- Finance Transaction

Observed error included:
- `POST /rest/v1/inventory_items ... 400 (Bad Request)`
- `organization_id and tenant_id cannot both be null`

**Root Cause:**  
ERP route pages (`InventoryPage`, `ProcurementPage`, `ProductionPage`, `FinancePage`) were using direct Supabase inserts without consistently stamping `organization_id` and `tenant_id`.  
As a result, row-level checks/constraints rejected create operations.

**Implemented Fixes:**
- Migrated ERP page-level data operations to module-scope context usage via `useModuleScope`.
- Added guarded context checks (`Organization context is required`) before mutations.
- Ensured create payloads include both:
  - `organization_id: organizationId`
  - `tenant_id: organizationId`
- Scoped read and destructive operations to current organization using `.eq("organization_id", organizationId)` to prevent cross-tenant access patterns.

**Files Updated:**
- `src/modules/erp/pages/InventoryPage.tsx`
- `src/modules/erp/pages/ProcurementPage.tsx`
- `src/modules/erp/pages/ProductionPage.tsx`
- `src/modules/erp/pages/FinancePage.tsx`

**Outcome:**  
ERP create flows now send organization-scoped payloads, resolving null-tenant/null-organization insert failures from these UI pages.

**Validation:**
- `npm run -s typecheck` executed; ERP page changes compile.
- Remaining typecheck errors are pre-existing unused-variable issues in unrelated modules.

**Verification:**

## 12. Bug Fix: Documentation Delete Failing on Missing `deleted_at` Column (Passed)

**Date:** 2026-04-12  
**Status:** PASSED

**Issue:**  
Deleting documents produced:
- `Error deleting document: Could not find the 'deleted_at' column of 'auto_documents' in the schema cache`

**Root Cause:**  
`useDeleteAutoDocument()` attempted soft delete via `softDeleteRecord(...)`, but `auto_documents` schema does not include soft-delete columns (`deleted_at`, `deleted_by`).

**Implemented Fixes:**
- Refactored `useDeleteAutoDocument()` to use organization-scoped hard delete for `auto_documents`.
- Added mutation-scope access verification before delete.
- Updated audit action to `document_deleted`.
- Updated success toast text to `Document deleted successfully`.

**Files Updated:**
- `src/modules/documents/hooks/useDocuments.ts`

**Outcome:**  
Document delete now works reliably for `auto_documents` in the current schema without soft-delete column errors.

**Validation:**
- `npx eslint src/modules/documents/hooks/useDocuments.ts src/modules/documents/pages/DocumentHistoryPage.tsx`

**Verification:**
