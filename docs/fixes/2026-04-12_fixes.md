## 2. Bug Fix: `dataset` Type Error in Reveal Observer (Passed)

**Date:** 2026-04-12  
**Status:** PASSED

**Issue:**  
TypeScript error: `Property 'dataset' does not exist on type 'Element'.`

**Root Cause:**  
`IntersectionObserverEntry.target` is typed as `Element`, so direct `dataset` access is not type-safe without narrowing.

**Implemented Fixes:**
- Added runtime/type narrowing (`entry.target instanceof HTMLElement`) before reading `dataset.revealId`.
- Kept observer behavior unchanged while making the code type-safe.

**Files Updated:**
- `src/workspaces/website/pages/Products.tsx`

**Outcome:**  
The observer logic remains functionally identical and compiles without the `dataset` typing error.

**Validation:**
- `npm run build`

**Verification:**
- Manually verified on April 12, 2026:
  - Confirmed reveal sections still animate in correctly.
  - Confirmed no `dataset` typing error remains.

## 7. Bug Fix: Owner Notification Click Staying in Owner Workspace (Passed)

**Date:** 2026-04-12  
**Status:** PASSED

**Issue:**  
When users were in the Organization Owner panel (`/organization/*`) and clicked a notification, they were navigated into tenant app routes (`/:tenantSlug/app/*`), which opened the Organization Admin workspace.

**Root Cause:**  
Notification links are generated as tenant app links for cross-workspace modules. The bell click and realtime toast actions navigated directly to those links without considering the current workspace context (owner panel vs tenant app panel).

**Implemented Fixes:**
- Added a shared notification navigation resolver that:
  - keeps existing UUID->slug normalization behavior,
  - detects when an `owner`/`admin` user is currently in `/organization/*`,
  - remaps tenant app notification links to owner workspace routes (e.g. `/organization/overview`, `/organization/users`, `/organization/subscription`, `/organization/alerts`) instead of jumping into `/:tenantSlug/app/*`.
- Updated both bell click handling and realtime toast `View` actions to use this resolver.

**Files Updated:**
- `src/core/utils/notification-links.ts`
- `src/ui/notification-bell.tsx`
- `src/core/hooks/useNotifications.ts`

**Validation:**
- `npx tsc --noEmit`

**Verification:**
- Manually verified on April 12, 2026:
  - Opened Organization Owner panel.
  - Clicked notification bell items and realtime toast `View` actions.
  - Confirmed navigation remains within `/organization/*` routes.
