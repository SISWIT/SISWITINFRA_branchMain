## 1. CRM Report 2.1 Broken Pipeline Drag-and-Drop

**Issue:**
Dragging an opportunity card to a different stage in the CRM pipeline visually snapped the card back to its original column, making the Kanban flow feel broken.

**Root Cause:**
`OpportunityPipeline` relied entirely on the parent query result and did not maintain any local optimistic board state during a drag-and-drop mutation. The UI therefore kept rendering the old stage until the server round-trip finished.

**Implemented Fixes:**

1. Added local pipeline state inside `OpportunityPipeline` so dropped cards move immediately in the board.
2. Synced that local board state back to the latest query result whenever fresh opportunities data arrives.
3. Added rollback behavior so the card returns to its previous stage only if the update mutation actually fails.
4. Persisted `is_closed` and `is_won` alongside the stage during drag-and-drop so closed-stage moves remain internally consistent.
5. Ignored drops back into the same stage to avoid unnecessary mutations and noisy success toasts.

**Files Updated:**

- `src/modules/crm/components/OpportunityPipeline.tsx`

**Outcome:**
Pipeline cards now stay in the destination column immediately after a valid drop instead of snapping back while the mutation is in flight.

## 2. CRM Report 2.2 Opportunities Data Persistence (Close Date)

**Issue:**
The Opportunities page could show `-` in the `Close Date` column even after a close date was entered during creation, and the edit form was reading from a row shape that did not match the shared CRM opportunity model.

**Root Cause:**

1. `OpportunitiesPage` maintained its own local opportunity row type and cast the query result into it, even though the shared CRM hook already returns a different `Opportunity` shape.
2. `useCRM.ts` stored opportunity notes in `next_step`, but did not expose that value through the shared `description` field the page was using.
3. The page relied on raw `new Date(...)` parsing instead of a normalized close-date parser, which made the display path more brittle than necessary.

**Implemented Fixes:**

1. Aligned `OpportunitiesPage` to use the shared `Opportunity` type returned by `useOpportunities()` instead of a local casted row interface.
2. Updated `mapOpportunity()` to expose `description` from the persisted `next_step` value so the page and hook stay in sync.
3. Added normalized close-date parsing for both the list column and the edit dialog prefill logic.
4. Kept the opportunities list and form bound to the same shared data model so persisted values are rendered consistently after refetch.

**Files Updated:**

- `src/modules/crm/hooks/useCRM.ts`
- `src/modules/crm/pages/OpportunitiesPage.tsx`

**Outcome:**
The Opportunities screen now reads and formats persisted close dates through the shared CRM opportunity model instead of a mismatched local row shape.

**Verification:**

## 7. CPQ Report 3.2 Add Product Selection / Account-Contact Stickiness (CPQ-004)

**Issue:**
The quote creation form could occasionally lose the first Account or Contact selection, making it feel like the choice did not "stick" until the user selected it again.

**Root Cause:**
The quote builder mixed direct snapshot-style object state writes with dependent CRM dropdowns that re-render when the selected account changes. That made the form more vulnerable to overwriting a just-selected value during quick re-renders.

3. Added stable `key` values to the dependent Contact and Opportunity selects so they fully reset when the account context changes.
4. Kept the existing account-change reset behavior for contact and opportunity selection, but moved it onto the same safe update path.

**Files Updated:**

## 8. CPQ Report 4.1 Templates Module Discrepancy (CPQ-003)

**Issue:**
CPQ exposed a `Templates` area, but it was only a discovery hub and not a real quote template feature. Users could not create reusable quote blueprints, save default line items, or apply a template directly into quote creation.

**Root Cause:**
CPQ was missing the core pieces of a real template workflow:

1. No persisted quote template data model in Supabase.
2. No shared hooks or types for quote templates and their default items.
3. No CRUD UI for managing reusable quote templates.
4. No quote-builder integration to apply template defaults into a live draft.
5. No clear employee-facing entry point to start quote creation from a template.

**Implemented Fixes:**

1. Added a real Supabase-backed quote templates schema with `quote_templates` and `quote_template_items` tables, indexes, and RLS policies.
2. Regenerated typed database definitions so the new CPQ template tables are available in the app type layer.
3. Added shared CPQ `QuoteTemplate` and `QuoteTemplateItem` types plus React Query hooks for listing, creating, updating, deleting, and reading template items.
4. Rebuilt `cpq/templates` into a true Quote Templates library with template cards, search, category filters, create/edit dialog, default line-item builder, and live pricing preview.
5. Kept the CPQ sidebar and dashboard template entry points, but now route them into the real template library instead of a placeholder hub.
6. Added `Use Template` navigation from the CPQ template library into quote creation with `?template_id=...`.
7. Added a new `Create From Template` action on the quotes list so sales users can reach reusable templates directly from their main quotes screen.
8. Added quote-builder template application flow that preloads validity, payment terms, customer notes, quote-level discount, GST, and default line items into a new draft without overwriting account/contact/opportunity context.
9. Added a template summary panel inside the quote builder so users can see what they are about to apply before launching the quote.

**Files Updated:**

## 9. CPQ Quote Templates UI Alignment & Quote Builder UX Refinement

**Issue:**
After the real Quote Templates feature was introduced, the new CPQ template and quote-builder screens felt visually heavier than the rest of the Employee Panel. The Create Quote page also felt crowded, and the summary/action area needed refinement for better day-to-day usability.

1. The first pass of the new Quote Templates UI used a more promotional hero-style presentation than the calmer card-based styling used in the rest of the CPQ and Employee Panel pages.
2. The initial Quote Builder template panel placed selected-template details beside the selector, which made the form feel wider and more fragmented than necessary.
3. The Create Quote form mixed commercial controls, notes, products, and actions in a way that made the page feel denser than the surrounding product screens.
4. Primary actions were initially placed in the top page header, which made them less useful once the user scrolled down into the quote form.
5. The first sticky-summary refinement introduced an unreliable sticky structure inside the existing dashboard scroll container, which made the summary move with the page more than expected.

**Implemented Fixes:**

1. Restyled `CPQTemplatesHubPage` to better match the overall site theme by replacing the heavy dark hero presentation with a standard Employee Panel page header, muted stat cards, softer surfaces, and cleaner template cards.
2. Simplified the templates search/filter bar and empty states so they feel consistent with the rest of the CPQ module instead of behaving like a separate microsite.
3. Refined the quote-template create/edit dialog styling so the form area and preview area use standard panel treatments and calmer contrast.
4. Reworked `QuoteBuilderPage` into clearer functional sections:
   - template selection
   - quote details
   - terms and notes
   - line items
   - quote summary
5. Reduced visual clutter in the Quote Builder by removing the more decorative background treatment and aligning the page to the site’s standard card-and-border system.
6. Moved the selected template information block to render below the template selector instead of beside it, so the section flows vertically and reads more naturally inside the form.
7. Changed the selected template metrics (`Items`, `Validity`, `Est. Total`) to display horizontally beneath the selected template details rather than in a narrow side stack.
8. Added a dedicated `Terms & Notes` section so payment terms and notes no longer compete visually with customer/account fields.
9. Reworked the line-item area so it reads more cleanly, with clearer empty state messaging, calmer item cards, and more consistent labels for quantity, price, discount, and totals.
10. Moved `Save Draft` / `Save Changes` and `Submit Quote` from the top page header into the Quote Summary area so users can act from the financial review panel while working through the form.
11. Added a more visible summary action container inside the Quote Summary card so the action area is easier to spot without feeling overly loud.
12. Reworked the Quote Summary sticky behavior by:
   - moving sticky positioning to the outer summary wrapper
   - capping the summary height on larger screens
   - letting the summary details scroll internally when needed
   - keeping the actions in a fixed footer region inside the summary card
13. Tuned the summary sidebar so totals, metadata, and actions display more reliably inside the Employee Panel dashboard layout instead of drifting with the page.

5. Removed the old `/user/mo` presentation from product cards.
6. Replaced that rigid suffix with a more generic `Standard quote price` helper line on the product cards.
7. Normalized the dialog copy so the product creation flow feels more professional and less tied to a single billing model.
