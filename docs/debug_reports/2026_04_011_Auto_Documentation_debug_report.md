## 3. Detailed Issue Logs

### [Issue #1] Sidebar Context Loss in Sub-routes
- **Title:** Sidebar Links Disappear in Sub-routes
- **Description:** When a user navigates from the Documents Dashboard to any sub-page (Manage Templates, History, Create Document), the "Documents" section in the sidebar disappears. This leaves the user with only CRM/CPQ links or a blank sidebar area, making navigation back to the dashboard difficult without browser "back" actions.
- **Steps to Reproduce:**
  1. Navigate to `/:tenantSlug/app/documents`.
  2. Click on "Manage Templates" or "Create Document".
  3. Observe the left sidebar navigation.
- **Expected Behavior:** The "Documents" section should remain visible and the active link should be highlighted.
- **Actual Behavior:** The "Documents" menu section is removed from the DOM or fails to render.
- **Severity:** High
- **Affected Page/Component:** `DashboardSidebar.tsx` / All Document sub-pages.

### [Issue #2] Hardcoded Legacy Routing Paths
- **Title:** Inconsistent Routing (Absolute vs. Dynamic)
- **Description:** Several action buttons and links within the Documents module use hardcoded absolute paths starting with `/dashboard/` instead of the required `/:tenantSlug/app/documents/` structure. This causes routing breaks when the app expects organization-scoped paths.
- **Steps to Reproduce:**
  1. Go to the Documents Dashboard.
  2. Hover over the "Quick Action" cards (e.g., Manage Templates).
  3. Notice the destination URL in the status bar (or click it).
- **Expected Behavior:** Links should use `tenantAppPath(tenantSlug, 'documents/...')`.
- **Actual Behavior:** Links point to `/dashboard/documents/...`.
- **Severity:** Medium
- **Affected Page/Component:** `DocumentsDashboard.tsx`

### [Issue #3] Performance Degradation (Skeleton Loop)
- **Title:** Excessive Page Load Times
- **Description:** Moving between pages in the module triggers skeleton loaders that persist for 5–10 seconds, even when the underlying data set is small (e.g., < 5 documents).
- **Steps to Reproduce:** Click between "Dashboard" and "History".
- **Expected Behavior:** Data should load in < 1 second.
- **Actual Behavior:** Prolonged "Loading documents..." state.
- **Severity:** Medium
- **Affected Page/Component:** Throughout `modules/documents`.
